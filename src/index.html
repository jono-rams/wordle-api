Wordle Game API Documentation

Welcome to the API documentation for the Wordle Game! This API provides a set of endpoints that allow you to generate random 5-letter words and check guesses against the hidden word, just like the popular Wordle game.

How to Play

  1. Start a new game: Make a GET request to /api/new-word to generate a new word and receive a unique session ID (uid).
  2. Submit guesses: Make POST requests to /api/guess, including the uid and your 5-letter guess in the request body.
  3. Get feedback: The API will respond with a result indicating which letters are correct and in the correct position (green), which letters are correct but in the wrong position (yellow), and which letters are not in the word (grey).
  4. Get the solution (if you give up): Make a GET request to /api/solution?uid=your-uid (replace your-uid with the actual session ID) to reveal the hidden word. Note: This will also delete the current game session, and you will no longer be able to make guesses for that uid.

Example Usage

# Start a new game
curl https://wordle-api.jono-rams.work/api/new-word

# Submit a guess (replace your-uid with the actual session ID)
curl -X POST -H "Content-Type: application/json" \
  -d '{"uid": "your-uid", "guess": "crane"}' \
  https://wordle-api.jono-rams.work/api/guess

Color Codes

The result array in the /api/guess response contains objects with a color property. These colors indicate the following:

  • green: The letter is in the correct position.
  • yellow: The letter is in the word but in the wrong position.
  • grey: The letter is not in the word.

Error Handling

The API may return the following error responses:

  • 400 Bad Request: Indicates an invalid request, such as an invalid session ID or an invalid guess format.
  • 500 Internal Server Error: Indicates an internal server error, such as a failure to fetch the word or store data.
